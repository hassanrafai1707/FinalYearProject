**Key Observations:**
1. All endpoints use `${app.version}` as the base path (e.g., `/api/v1/...`)
2. Most endpoints use request bodies instead of path variables
3. Common patterns: GET for retrieval, POST for creation, PATCH for updates, DELETE for deletion
4. Admin endpoints require admin password for destructive operations
5. Question paper generation is in Teacher controller
6. Question paper approval workflow is in Supervisor controller
7. All roles have self-service email/password update endpoints
8. Student and Supervisor have overlapping question retrieval endpoints
